     Plan: Add Bazaar discovery extension to GET /skills/:id

     Context

     The Bazaar has no POST registration API. Resources are auto-indexed by the
     facilitator (CDP) during payment verification — the server embeds discovery
      metadata in the 402 payment response via declareDiscoveryExtension, and
     the facilitator picks it up when settling payments.

     POST /skills/generate already has declareDiscoveryExtension (added in
     previous session).
     GET /skills/:id does not — meaning skill URLs are missing discovery
     metadata.

     Pre-existing skills (already in S3 when the server starts) will be
     auto-indexed by the facilitator the first time a client pays to access
     them, but only if the GET /:id route includes the discovery extension.
     Without it, the 402 response for skill downloads carries no Bazaar
     metadata, so the facilitator cannot catalog those URLs.

     There is no mechanism to proactively push existing skills into the Bazaar —
      the auto-indexing happens organically on first paid access. Adding the
     extension to GET /:id ensures that when any skill is accessed (whether
     newly generated or pre-existing), the facilitator receives the discovery
     schema.

     Change

     File: src/routes/skills.js

     Add declareDiscoveryExtension to the GET /:id route config:

     'GET /:id': {
       accepts: {
         scheme: 'exact',
         price: '$0.01',
         network: NETWORK,
         payTo: payToAddress,
       },
       description: 'Download a generated Agent Skill ($0.01 USDC)',
       ...declareDiscoveryExtension({}),   // query-style GET, no input params
     },

     declareDiscoveryExtension({}) with no bodyType creates a query-style
     extension (HTTP GET), which is correct for this endpoint.

     Verification

     1. Start the server (npm start or npm run dev)
     2. Confirm no Bazaar-related warnings on startup
     3. Make a GET request to /skills/:id without payment — should get a 402
     with the x402 payment requirements; the response body or headers should
     include Bazaar discovery extension metadata
     4. Optionally use the demo script to list Bazaar resources after making a
     paid request
