‚óŹ Given the project uses Claude on Node.js with a relatively simple 3-stage
  pipeline, here are the practical options:

  1. Direct Anthropic SDK (@anthropic-ai/sdk)

  - Tool use via messages API, you write the agent loop yourself
  - Full control, minimal abstraction, no extra dependencies
  - Good fit when the agent logic is straightforward (Stage 1 tool loop is
  simple)

  2. Vercel AI SDK (ai + @ai-sdk/anthropic)

  - generateText() with tools, automatic tool call execution, streaming
  - Provider-agnostic (easy to swap models later)
  - Nice DX but adds an abstraction layer you may not need

  3. Anthropic Agent SDK (@anthropic-ai/agent-sdk)

  - Purpose-built for Claude agent loops with tool use
  - Newer, less community adoption ‚ÄĒ worth checking current state

  4. Mastra

  - Full agent framework with tool management, workflows, memory
  - Heavier ‚ÄĒ likely overkill for a 3-stage pipeline

  My recommendation: Option 1 (direct SDK) or Option 2 (Vercel AI SDK). The
  pipeline is simple enough that a full agent framework adds more weight than
  value. The direct SDK keeps things minimal; the Vercel AI SDK adds convenience
   for tool execution loops without much overhead.

  Want me to research any of these in more depth, or do you have a preference?
